# Task 04 - Use Copilot Chat context, extensions, and commands

## Introduction

GitHub Copilot Chat provides three powerful ways to focus and enrich your prompts:

| Prefix | Feature | Purpose |
|--------|---------|---------|
| `#` | **Context variables** | Attach specific files, symbols, selections, or workspace artifacts to a prompt |
| `@` | **Extensions / participants** | Route a prompt to a scoped agent that has deep knowledge of a subsystem |
| `/` | **Commands** | Trigger a predefined Copilot action (explain, fix, generate tests, etc.) |

In this task you will practise each feature against the Zava Storefront solution.

## Description

Use context variables, extensions, and slash commands in Copilot Chat to explore, explain, and improve the Zava Storefront source code.

## Success Criteria

- You have used at least one `#` context variable, one `@` extension, and one `/` command in Copilot Chat.
- You have used Copilot to explain a file, generate tests, and fix a warning in the solution.

## Key Tasks

### 01: Use context variables (`#`) to scope your prompt

Context variables let you pin a specific file, symbol, selection, editor content, or terminal output directly into your prompt so Copilot answers with precision.

**Common context variables**

| Variable | What it attaches |
|----------|-----------------|
| `#file` | A specific file from the workspace |
| `#selection` | The currently highlighted code in the editor |
| `#codebase` | The entire indexed workspace |
| `#terminalLastCommand` | The last command run in the integrated terminal |
| `#terminalSelection` | Selected text in the integrated terminal |

<details markdown="block">
<summary><strong>Expand this section for detailed steps</strong></summary>

1. Open Copilot Chat in VS Code (``Ctrl+Alt+I`` or the chat icon in the Activity Bar).

2. Attach the `ProductService.cs` file as context using the `#file` variable and ask Copilot to summarise it:

    ```text
    #file:ProductService.cs Summarise what this service does and list its public methods.
    ```

3. Open `CartController.cs` in the editor, select a method body, then reference your selection:

    ```text
    #selection Explain this method step by step and identify any potential bugs.
    ```

4. After a failed `dotnet build`, reference the last terminal command to let Copilot diagnose the output:

    ```text
    #terminalLastCommand What went wrong and how should I fix it?
    ```

</details>

### 02: Use extensions / participants (`@`) to route prompts

Extensions (also called *chat participants*) give Copilot specialised knowledge about a subsystem. Prefix your message with `@<participant>` to direct the question to the right expert.

**Built-in participants**

| Participant | Expertise |
|-------------|-----------|
| `@workspace` | Your local codebase – file structure, symbols, cross-file references |
| `@vscode` | VS Code settings, extensions, keyboard shortcuts, commands |
| `@terminal` | The integrated terminal – shell commands and their output |
| `@github` | Your GitHub repositories, issues, pull requests, and code search |

<details markdown="block">
<summary><strong>Expand this section for detailed steps</strong></summary>

1. Ask `@workspace` to find where products are loaded into the cart:

    ```text
    @workspace Where in the codebase are products loaded into the cart? Show me the relevant files and methods.
    ```

2. Ask `@workspace` to understand the overall architecture:

    ```text
    @workspace Give me an overview of the solution architecture, listing the key controllers, services, and models.
    ```

3. Use `@vscode` to discover a productivity shortcut:

    ```text
    @vscode What keyboard shortcut opens the integrated terminal?
    ```

4. Use `@terminal` to get help constructing a shell command:

    ```text
    @terminal How do I run only the unit tests in the ZavaStorefront solution from the terminal?
    ```

5. Use `@github` to check for open issues related to the .NET upgrade:

    ```text
    @github Show me any open issues in this repository related to .NET version upgrade.
    ```

</details>

### 03: Use slash commands (`/`) for predefined actions

Slash commands are shortcuts that trigger a specific Copilot behaviour without writing a long prompt.

**Common slash commands**

| Command | Action |
|---------|--------|
| `/explain` | Explain the selected code or attached file |
| `/fix` | Propose a fix for the selected code or a compiler error |
| `/tests` | Generate unit tests for the selected code |
| `/doc` | Generate documentation comments for the selected code |
| `/new` | Scaffold a new file, class, or project |
| `/clear` | Clear the current chat session |

<details markdown="block">
<summary><strong>Expand this section for detailed steps</strong></summary>

1. Open `CartService.cs`, select the entire class body, then run:

    ```text
    /explain
    ```

    Copilot will walk through the class logic, explaining each method.

2. Select the `Checkout` method in `CartController.cs` and generate unit tests:

    ```text
    /tests
    ```

    Review the suggested tests and, if satisfied, use the **Insert into new file** button to save them.

3. Select a method that has no XML documentation comment and generate one:

    ```text
    /doc
    ```

4. If your earlier build produced a compiler warning, select the warning line in the terminal or editor and run:

    ```text
    /fix
    ```

    Copilot will propose a targeted code change to resolve the warning.

5. Scaffold a new empty service class:

    ```text
    /new Create a new C# service class called WishlistService in the Services folder.
    ```

</details>

### 04: Combine `#`, `@`, and `/` in a single prompt

The real power comes from combining all three features.

<details markdown="block">
<summary><strong>Expand this section for detailed steps</strong></summary>

1. In Copilot Chat, type the following combined prompt:

    ```text
    @workspace /tests #file:CartService.cs Generate xUnit tests that cover the AddItem and RemoveItem methods, using Moq for any dependencies.
    ```

    This routes the request to the workspace participant (`@workspace`), uses the `/tests` command as guidance, and pins `CartService.cs` as the exact source (`#file`).

2. Review Copilot's response and insert the generated tests into a new file.

3. Build the solution to confirm the tests compile:

    ```bash
    dotnet build
    ```

</details>
